# SCLAB Vision Stand-alone Docker Deployment

This directory runs SCLAB Vision without the root SCLAB Docker stack.

The default stack is:

- `mongo`
- `redis`
- `qdrant`
- `vision-aio`
- `vision-console`
- `vision-aio-tls` and `vision-console-tls` HTTPS proxies

Container names and the Docker network are prefixed with `vision-stand-alone` to avoid conflicts with the root `docker-compose.yml`.

## Quick Install

```sh
./install.sh
```

For unattended default installation:

```sh
./install.sh -y
```

The installer will:

1. Check Docker and Docker Compose, and offer to install Docker on common Linux distributions.
2. Ask setup questions and generate `.env`.
3. Create local directories under `./data/`.
4. Generate a self-signed TLS certificate in `./data/vision/certs/` when `cert.pem` and `privkey.pem` are missing.
5. Log in to ECR when possible, pull images, and run `docker compose up -d`.

## Operations

```sh
./up.sh        # start
./down.sh      # stop and remove containers; ./data remains
./logs.sh      # follow logs, e.g. ./logs.sh vision-aio
./restart.sh   # restart
./pull.sh      # pull images
./update.sh    # pull + recreate
```

## URLs

- Console web UI: `https://<host>:8890`
- Control API: `https://<host>:8090`
- HLS gateway: `https://<host>:8080`

## Layout

- Network: `vision-stand-alone-network`, created by Docker Compose.
- Local data:
  - `data/mongo/`
  - `data/redis/`
  - `data/qdrant/`
  - `data/vision/recordings/`
  - `data/vision/certs/` for TLS certificate files (`cert.pem`, `privkey.pem`)
  - `data/vision/rustfs/` when the `s3` profile is enabled
- Config: `.env`, generated by `install.sh`.
- Compose profiles: `s3` only. GPU mode uses `COMPOSE_FILE=docker-compose.yml:docker-compose.gpu.yml`.

The default development secrets are unsafe for production. `install.sh` generates random secrets by default.
