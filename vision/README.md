# SCLAB Vision Docker Deployment

This directory installs SCLAB Vision with Docker Compose. The default stack is:

- `vision-aio`
- `vision-console`
- the root stack `sclab-proxy` HTTPS proxy
- existing `mongo`, `redis`, and `qdrant` containers on `sclab-network`

## Quick Install

```sh
./install.sh
```

The installer asks step-by-step questions. Press Enter at every prompt to use the defaults.

For unattended default installation:

```sh
./install.sh -y
```

The installer will:

1. Check Docker and Docker Compose, and offer to install Docker on common Linux distributions.
2. Create `sclab-network` if it does not exist.
3. Ask six setup sections and generate `.env`.
4. Create local directories under `./data/vision/`.
5. Log in to ECR when possible, pull images, and run `docker compose up -d`.

## Installer Questions

| Step | Setting | Default |
|---|---|---|
| 1 Accelerator | NVIDIA GPU support | CPU |
| 2 Shared services | MongoDB, Redis, Qdrant endpoints | existing `mongo` / `redis` / `qdrant` on `sclab-network` |
| 3 Recording | disabled, disk, or disk + S3 | disabled |
| 4 Main HTTPS proxy ports | console 8890, gateway 8080, control 8090 | fixed in the root stack |
| 5 Images | registry and tag | ECR, `0.1.1` |
| 6 Secrets | aio/console shared secrets | auto-generated |

## Operations

```sh
./up.sh        # start
./down.sh      # stop and remove containers; ./data/vision data remains
./logs.sh      # follow logs, e.g. ./logs.sh vision-aio
./restart.sh   # restart
./pull.sh      # pull images
./update.sh    # pull + recreate
```

## Layout

- Network: `sclab-network` external network, shared with the main SCLAB stack.
- Local data: all Vision-owned files are under `./data/vision/`.
  - `recordings/` for DVR hot segments.
  - `rustfs/` for optional S3-compatible cold-tier recording.
- Shared data services: MongoDB, Redis, and Qdrant are not started by this compose file. They must already be running as `mongo`, `redis`, and `qdrant` on `sclab-network`.
- HTTPS proxy: the root `sclab-proxy` container exposes Vision HTTPS ports and uses the root `./cert/cert.pem` and `./cert/privkey.pem` files.
- Config: `.env`, generated by `install.sh`. Edit it and run `./up.sh` to apply changes.
- Compose profiles: `s3` only. GPU mode uses `COMPOSE_FILE=docker-compose.yml:docker-compose.gpu.yml`.

## URLs

- Console web UI: `https://<host>:8890`
- Control API: `https://<host>:8090`
- HLS gateway: `https://<host>:8080`

## Shared Service Defaults

Defaults match the sibling SCLAB Docker stack:

```env
VISION_MONGO_URL=mongodb://root:changeThisMongoPassword@mongo:27017/?authSource=admin
VISION_MONGO_DB=sclab
VISION_REDIS_URL=redis://:changeThisRedisPassword@redis:6379
VISION_QDRANT_URL=http://qdrant:6333
VISION_QDRANT_API_KEY=changeThisQdrantApiKey
```

Change these values in `.env` if the main stack uses different passwords or service names.

The default development secrets (`sv-dev-*`) are unsafe for production. `install.sh` generates random secrets by default.
